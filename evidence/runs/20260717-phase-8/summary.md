# Phase 8 read-only assistant evidence

Phase 8 is complete. The implementation commit under test was
`ac8a43fc8dd975198546b00eb09416d2cd9ecad0`. The optional assistant remained absent from the
default FleetPulse stack and used a deterministic offline provider for reproducible verification.

## Measured local results

| --- | --- |
| Golden safety/accuracy set | 5/5 cases passed (100% deterministic pass rate) |
| Assistant unit/API tests | 7/7 passed |
| Full repository tests | 28/28 passed |

The golden set exercised a supported cited claim, secret redaction, an invented evidence ID that
forced safe abstention, empty-evidence abstention, and an instruction embedded in untrusted
evidence. Every case also checked read-only authority, citation resolution, output-secret absence,
and remediation approval gating.

## Acceptance criteria

- The default Compose profile has no assistant or AI dependency.
- Redaction occurs before provider access and again before output validation.
- Non-abstaining claims and proposals cite only supplied evidence IDs.
- Invalid provider output or unknown citations fail closed into abstention with no proposal.
- Review requires a proposal generated by the process, duplicate review is rejected, and approval
  exposes no execution capability.
- The optional service is non-root, loopback-only, edge-only, and receives no database, Redis,
  agent, Kubernetes, shell, or deployment credential.
- Structured correlation IDs were preserved across readiness, analysis, and review requests.

## Evidence map

- `accuracy.json`: per-case checks and deterministic results.
- `container-smoke.json`: redaction, citation, review, authority, port, network, user, and credential
  assertions from the optional container.
- `metadata.json`: implementation SHA, environment, provider, classification, and commands.
- `verification.txt`: full verification, secret scan, boundary check, and known limitations.

## Claims and limitations

This evidence validates FleetPulse's local application safety contract. It does not measure a live
model, general incident-diagnosis quality, production load, cloud behavior, or provider uptime.
The OpenAI adapter follows the official Responses API structured-output contract but was not
called because Phase 8 requires no external key and AI cannot be a production dependency.

Review receipts are intentionally non-executing and stored only in a bounded in-memory registry.
A future durable multi-replica approval audit would need isolated storage and coordination while
preserving the same no-execution boundary.
